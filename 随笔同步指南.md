## 📋 三种同步方案

### 方案A：手动同步（推荐新手）
**优点**: 完全控制，简单直接
**适用**: 刚开始使用，不熟悉技术操作

#### 操作步骤：
1. **编写随笔**：在 `/home/admin/.openclaw/workspace/memory/` 目录下按日期记录
2. **转换为网页**：手动将markdown格式转换为HTML
3. **更新网站**：使用下面的命令

```bash
# 进入网站目录
cd ~/fengyun-podcast

# 使用脚本创建模板（推荐）
./add-essay.sh "今日感悟" "2026-04-01"

# 编辑内容
vim essays.html

# 提交并推送
git add .
git commit -m "添加新随笔：今日感悟"
git push origin master
```

### 方案C：自动化同步（高级用户）
**优点**: 全自动化，设置一次即可
**适用**: 技术熟练用户，希望完全自动化

#### 方法1：使用Git Hook
```bash
# 1. 创建post-commit hook
cat > .git/hooks/post-commit << 'EOF'
#!/bin/bash
# 自动推送到GitHub
git push origin master
EOF

# 2. 添加执行权限
chmod +x .git/hooks/post-commit
```

#### 方法2：创建自动化脚本
```bash
# 创建自动化脚本
cat > auto-sync.sh << 'EOF'
#!/bin/bash

# 风云播客自动化同步脚本

# 设置目录
WEBSITE_DIR="$HOME/fengyun-podcast"
MEMORY_DIR="$HOME/.openclaw/workspace/memory"

# 检查今天的随笔文件是否存在
TODAY=$(date +%Y-%m-%d)
MEMORY_FILE="$MEMORY_DIR/$TODAY.md"

if [ ! -f "$MEMORY_FILE" ]; then
    echo "❌ 今天的随笔文件不存在: $MEMORY_FILE"
    exit 1
fi

echo "✅ 找到今天的随笔文件: $MEMORY_FILE"

# 进入网站目录
cd "$WEBSITE_DIR"

# 创建HTML格式的随笔
HTML_CONTENT=$(cat << EOF
        <article class="essay">
            <h2>今日随笔</h2>
            <p class="date">$TODAY</p>
            $(sed 's/^/            <p>/' "$MEMORY_FILE" | sed 's/$/<\/p>/')
        </article>
EOF
)

# 备份原文件
cp essays.html essays.html.backup

# 在第一个随笔前插入新随笔
sed -i "/        <article class=\"essay\">/i\\$HTML_CONTENT" essays.html

# 提交并推送
git add .
git commit -m "自动同步今日随笔: $TODAY"
git push origin master

echo "🎉 随笔同步完成！"
EOF

# 添加执行权限
chmod +x auto-sync.sh
```

#### 使用自动化脚本：
```bash
# 每天随笔写完后，运行：
./auto-sync.sh
```

## 🛠️ HTML格式参考

### 基本格式
```html
<article class="essay">
    <h2>随笔标题</h2>
    <p class="date">2026-04-01</p>
    <p>第一段内容...</p>
    <p>第二段内容...</p>
    <p><strong>加粗文字</strong></p>
    <p><em>斜体文字</em></p>
</article>
```

### 高级格式
```html
<article class="essay">
    <h2>随笔标题</h2>
    <p class="date">2026-04-01</p>
    
    <h3>小标题</h3>
    <p>段落内容...</p>
    
    <div class="quote">
        <p>引用内容...</p>
    </div>
    
    <p>普通段落 <span class="highlight">高亮文字</span> 继续内容...</p>
</article>
```

## 🔧 常见问题

### Q: 如何修改已发布的随笔？
A: 直接编辑 `essays.html` 文件，找到对应的随笔部分修改，然后提交推送。

### Q: 如何删除随笔？
A: 在 `essays.html` 文件中找到对应的 `<article>...</article>` 部分，整块删除，然后提交推送。

### Q: HTML格式写错了怎么办？
A: 网站会尽力显示，但建议使用上面提供的标准格式。可以用浏览器验证效果。

### Q: 如何备份？
A: Git本身就是最好的备份。所有历史版本都保存在GitHub上。
